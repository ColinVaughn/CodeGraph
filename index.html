// what it does

More than search.
It understands structure.

Every inferred relationship is tagged EXTRACTED, INFERRED, or AMBIGUOUS — so you can audit what the graph believes.

Graph & query

Relevance-ranked subgraph search, shortest paths, and node explanations over a graph of symbols and relationships in 30+ languages.

$ synaptic query "session reaper"

Reverse impact & foresight

Trace what a change would break before you make it — then predict the blast radius and speculate by actually running the at-risk tests in a throwaway worktree.

$ synaptic affected parse_config

Dynamic-dispatch awareness

Event buses, Electron IPC, reflection and dispatch tables are modeled or cataloged — so a "0 dependents" answer is never mistaken for "safe to change."

$ synaptic hazards

Safe refactor

Rename, move, or extract emits a confidence-scored plan for an agent to apply — then verify rebuilds and checks the graph held. Synaptic never edits your source.

$ synaptic refactor rename

SQL audit

Flags row-level-security gaps, over-broad grants, likely injection, missing indexes, SELECT * and N+1 patterns over a SQL-aware graph that links queries to the tables they touch.

$ synaptic sql audit

MCP server

28 read-only tools over stdio or HTTP, so your AI assistant queries the graph — navigation, impact, PR blast radius, SQL audit — before it greps or reads a single file.

$ synaptic serve

// install

One command to a full graph.

Builds with a stable Rust toolchain, or grab a prebuilt binary for Linux, macOS, or Windows.

install
# from a clone — installs the `synaptic` binary onto PATH
$cargo install --path bin/synaptic
 
# ...or build it in-tree
$cargo build --release → target/release/synaptic
 
# prebuilt binaries: Linux · macOS · Windows → Releases
synaptic extract .
build the graph → synaptic-out/
synaptic query "authentication flow"
ask the graph a question
synaptic affected parse_config
what would changing this break?
synaptic serve
hand the graph to your AI assistant over MCP
